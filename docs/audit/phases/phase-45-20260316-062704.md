## Main architecture conclusions

- the active deployable platform is the `rasid-platform-core` monorepo, with `apps/contracts-cli/src/dashboard-web.ts` acting as the current unified gateway
- the architecture is contract-first, with `packages/contracts` defining shared schemas and registries used by the engine packages
- most active runtime state is persisted as filesystem-backed JSON/artifact bundles under `.runtime/*`
- `apps/rasid-web` is architecturally important but currently separate from the deployed monorepo gateway, because it has its own Express/tRPC server, JWT auth, sql.js store, optional MySQL schema, and direct AI/storage integrations
